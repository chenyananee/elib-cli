# cli list

1. 所有资源由用户分配

2. 历史记录和接收缓存只使用两个一维数组

3. tab可匹配指令提示

4. 退格可删除接收字符

5. 上下键可选择历史记录

6. 指令注册以及回调注册接口

7. 初始化只初始化资源，包含换行的类型

8. 用户只需要在主循环中调用一个接口，用户需要给出tick周期（ms）

9. 回显以及信息打印

## 实现细节

1. 关于缓存公用问题，如果已经记录了2条指令如：cmd1\ncmd2\n则新指令在cmd2\n的后面如果缓存不够了则清除最早的一条指令如cmd2\ncmd3\ncmd4

2. 关于tab匹配，如果匹配到1条则直接补全指令，如果匹配到多条则输出匹配的所有指令需要用户进一步输入

3. 关于历史选择，遍历历史缓存从后向前第一个\n到第二个\n的内容就是上一条，以此类推，按下键最后一条就是用户已输入的内容或者最后一个\n之前的内容

4. 对于历史缓存覆盖的问题，例如已有历史cmd1\ncmd2\ncmd3\n，这时用户输入了有效的cmd4且历史缓存不够了但是占用比cmd1长，则需要清除cmd1和cmd2，现在历史缓存应该是cmd3\ncmd4\n

5. 关于回显以及指令信息打印，用户需要注册print接口printf类似，cli内部不做任何缓存以及格式转换
